Features of Website

The website is built using the Flask framework. Here are features of this website.
Happy reading!

Website details

  • Welcome: this will be visible on the top of all pages above nav bar
    • When the user is not logged in, it will display Welcome!
    • When the user is logged in, it will display Welcome <username>! Where <username> is the name entered during registration by the user
  1. Nav Bar
    • This is home
    • All blogs
    • Blog details
    • Register/Update Details
    • Login/Logout (shows "Logout" if the user is logged in, otherwise "Login")
    • Create new post (only visible to admin when logged in)
    • User details (only visible to logged-in users)
    • About me
    • Social media icons
       
  2. Contact form (visible all times at the bottom of page)
    • Send message (only logged in user can send message to admin)
       
  3. This is Home
    • Display latest post at the top
    • Only latest six post will appear on home page
    • “More” button will open pages where all blogs appear (linked with all blogs)
    • “Full post” button (linked with blog details, visible all time)
    • Clickable titles and images (linked with blog details)
    • Edit/Delete button (only visible to admin when logged in)
    • Create new post button at the bottom (only visible to admin when logged in)
    • Delete post:
      1. Admin need to follow an addition step before finally deleting the post
      2. Confirmation page shows post id and title details and asks user whether to delete or cancel
      3. If user finally chooses to delete the post, will be deleted from database
         
  4. All blogs
    • Show all blogs in database (starting from latest to oldest)
    • Clickable title and image (linked with blog details)
    • Full Post button (visible with each blog at all time)
    • Edit post/full post/delete post (only visible to admin with each blog, when logged in)
    • Create new post/ go up/ back to home buttons at bottom (create button only visible to admin when logged in, other visible all time)
       
  5. Blog Details (not clickable on nav bar)
    • Displays details of the selected post
    • Back to home button (returns to home page)
    • Edit post button (only visible to admin when logged in)
       
  6. Register/ Update Details (visible only one at a time to all users, default: register)
    • register
    • Takes username, email and password from user
    • If email does not exist in database, it will allow user to register first and after than open login page
    • “Already register Login!”  Button visible only if user not logged in
    • Update details (appear when user decides to update his details)
      • User can change username, email and password
      • Update me and cancel button at the bottom
      • If entered email already exists in database other than its own, it will ask user to choose other email. User can change his username and password without changing email.
         
  7. Login/Logout
    • Appear only one at a time (default login)
    • If user already logged in it will show logout, otherwise login
    • logout takes user to home page after logged out
       
  8. Create new post/Edit post (only visible to admin when logged in)
    • Edit post
      • “Edit post” written on top as heading
      • All form fields prefilled with existing data
      • Data will be updated after submission, and blog details page will be opened
    • Create new post (default)
      • “New post” written on the top as heading
      • Blank form fields will appear
      • After submitting details, data will be added on database
         
  9. User Detail
    • Visible only to logged in users
    • Page will show username and email address
    • Update/ delete/ back to home button at bottom
    • Update button will update user details (point-6(b))
    • Delete account button can delete user account
      • If the user wants to delete their account, they will need to confirm their password first
      • user will be taken to confirm delete page where they can choose either delete or cancel
      • if user decides to delete their account, they will need to enter their password
      • Password is necessary to delete the account and will be matched with the password in database
      • If password matches, user will be logged out and their details will be deleted from database
      • Admin is not allowed to delete their account from database, they can only update information
         
  10. About me
    • Static page with a back to home button at bottom
